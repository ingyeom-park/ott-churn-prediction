지도 학습이란?

지도 학습(Supervised Learning)은 입력 데이터와 그에 대응하는 원하는 출력이 함께 주어진 상태에서 모델을 학습시키는 머신러닝 방식입니다. 입력/출력 쌍(input/output pairs)으로 이루어진 훈련 세트에서 모델을 만드는 과정.

예를 들어 어떤 이메일이 주어졌을 때 그 이메일이 스팸인지 아닌지를 알려주는 데이터가 있다면, 모델은 입력인 이메일과 출력인 정답 라벨 사이의 관계를 학습합니다.

여기서 중요한 점은 모델이 단순히 훈련 데이터의 정답을 외우는 것이 아니라는 점입니다. 학생이 기출문제의 답 번호만 외우면 새 문제를 풀 수 없듯이, 지도 학습 모델도 이미 본 데이터만 맞혀서는 충분하지 않습니다. 목표는 훈련 세트에서 배운 관계를 바탕으로, 이전에 본 적 없는 새로운 입력에 대해서도 정확한 출력을 예측하는 것입니다.

지도 학습 문제는 출력의 성격에 따라 크게 두 종류로 나뉩니다. 출력이 미리 정해진 선택지 중 하나의 클래스 라벨(class label)이라면 분류 (classification) 문제이고, 출력이 연속적인 숫자값이라면 회귀 (regression)문제입니다. 즉 분류와 회귀의 차이는 사용하는 알고리즘 이름보다 먼저, 우리가 예측하려는 출력이 “범주”인지 “수치”인지에서 출발합니다.

대표적인 예는 스팸 필터입니다. 여기서 입력은 이메일의 내용이고, 출력은 “스팸” 또는 “스팸 아님”이라는 두 개의 라벨 중 하나입니다. 이 문제는 결국 “이 이메일은 스팸인가?”라는 예/아니오 질문으로 바꿀 수 있습니다. 따라서 스팸 필터는 두 클래스 중 하나를 고르는 이진 분류(binary classification)의 예가 됩니다.

모델 : X (입력) → ŷ (출력 예측)
02 / 05

분류 vs 회귀

분류(classification) 문제의 목표는 미리 정해진 클래스 목록 중 하나의 라벨을 예측하는 것입니다. 여기서 클래스란 모델이 고를 수 있는 선택지입니다.

이진 분류에서는 두 클래스 중 하나를 양성 클래스(positive class), 다른 하나를 음성 클래스(negative class)라고 부르기도 합니다. 여기서 positive는 “좋다”는 뜻이 아닙니다. 연구자가 관심을 두고 탐지하려는 대상이라는 뜻에 가깝습니다. 스팸 탐지에서는 스팸이 positive class가 될 수 있고, 질병 진단에서는 질병이 있는 경우가 positive class가 될 수 있습니다. 즉 positive와 negative는 가치판단이 아니라 분석의 초점에 따라 정해지는 이름표입니다.

분류 (Classification)

출력 = 이산적 카테고리

정해진 클래스 목록 중 하나를 예측합니다. 클래스 사이에 연속성이 없습니다. 붓꽃 품종 예측처럼 클래스가 세 개 이상인 경우는 다중 클래스 분류(multiclass classification)라 합니다.

예: 종양 양성/악성, 스팸 분류, 웹사이트 언어 판별

빈 곳을 클릭하면 해당 위치의 예측 클래스와 가장 가까운 이웃이 표시됩니다

회귀(regression) 문제의 목표는 연속적인 숫자값을 예측하는 것입니다. 교재는 이를 프로그래밍 관점에서는 부동소수점(floating-point number), 수학적으로는 실수(real number)를 예측하는 문제라고 설명합니다. 예를 들어 학력, 나이, 거주 지역을 바탕으로 연간 소득을 예측하는 문제는 회귀입니다. 이때 모델이 예측하는 것은 “상위/중위/하위” 같은 범주가 아니라 실제 금액이라는 양(amount)입니다.

또 다른 예로, 이전 수확량, 날씨, 투입된 인력 등의 정보를 바탕으로 옥수수 농장의 수확량을 예측하는 문제도 회귀입니다. 수확량은 정해진 몇 개의 클래스 중 하나가 아니라, 일정 범위 안에서 다양한 값을 가질 수 있는 연속적인 출력입니다.

회귀 (Regression)

출력 = 연속적 실수

예측값이 연속된 수치입니다. 회귀 과제에서 예측값은 양(amount)이며, 주어진 범위 안의 어떤 값도 될 수 있습니다. 옥수수 농장의 이전 수확량·날씨·인력으로 수확량을 예측하는 것도 회귀 문제입니다.

예: 연간 소득 예측, 주택 가격, 옥수수 수확량

빈 곳을 클릭해 점을 추가하면 회귀선과 R²가 바뀝니다

가장 쉬운 구분법: 출력에 연속성(continuity)이 있는지 물어보는 것입니다. 연소득 예측에서 $40,000과 $40,001의 차이는 거의 없습니다. 반면 웹사이트의 언어를 판별하는 문제에서는 정도의 차이가 없습니다. 영어이거나 프랑스어이거나 둘 중 하나이고, 영어와 프랑스어 사이 어딘가에 있는 언어란 존재하지 않습니다.

일반화 (Generalization)

지도 학습에서 우리가 진짜 원하는 것은, 훈련 데이터로 만든 모델이 훈련 세트와 동일한 특성을 가진 새로운 데이터에 대해서도 정확한 예측을 내놓는 것입니다. 모델이 처음 보는 데이터에 대해 정확한 예측을 한다면, 훈련 세트에서 테스트 세트로 일반화(generalize)되었다고 말합니다. 우리의 목표는 최대한 정확하게 일반화할 수 있는 모델을 만드는 것입니다.

🎓

훈련 세트 (Train Set)

알고리즘이 학습에 사용하는 데이터. 입력-출력 쌍에서 패턴을 추출합니다.

🔬

테스트 세트 (Test Set)

모델이 한 번도 본 적 없는 데이터. 실제 일반화 성능은 여기서만 측정할 수 있습니다.

🎯

일반화 성능

테스트 세트에서의 정확도. 이것이 낮으면 훈련 성능이 아무리 높아도 의미가 없습니다.

보통은 모델이 너무 복잡하면 훈련 데이터에 과적합(overfitting)된다고 말합니다. 과적합된 모델은 훈련 세트의 각 데이터 포인트에 지나치게 최적화된 나머지, 새 데이터에는 제대로 작동하지 않습니다. 반대로 모델이 너무 단순하면 과소적합(underfitting)이 발생합니다.

모델이 복잡할수록 훈련 데이터를 더 잘 예측할 수 있습니다. 그러나 특정 수준 이상으로 복잡해지면 각 데이터 포인트의 개별적인 특성에 지나치게 집중하게 되어, 훈련 세트를 대표하는 패턴을 더 이상 포착하지 못합니다.

04 / 05

과적합 & 과소적합

훈련 데이터에서 더 복잡한 모델을 만들수록, 일반적으로 훈련 정확도는 높아집니다. 하지만 이것이 새 데이터에 대한 성능과 항상 일치하지는 않습니다. 모델 복잡도와 성능의 관계는 아래 그림처럼 나타납니다.

📉

과소적합 (Underfitting)

모델이 너무 단순해서 훈련 데이터의 다양성을 포착하지 못한 상태입니다. 훈련 성능과 테스트 성능이 모두 낮습니다.

✅

스위트 스팟 (Sweet Spot)

복잡도가 적절한 모델로, 훈련 데이터를 잘 학습하면서도 처음 보는 데이터에 일반화도 잘 됩니다.

📈

과적합 (Overfitting)

모델이 훈련 세트에만 지나치게 최적화된 상태입니다. 훈련 성능은 높지만, 처음 보는 데이터에는 잘 작동하지 않습니다.

모델 복잡도와 데이터셋 크기의 관계

모델 복잡도는 훈련 데이터셋에 포함된 입력의 다양성(variation)과 밀접하게 연결되어 있습니다. 데이터셋이 다양한 데이터 포인트를 많이 포함할수록, 과적합 없이 더 복잡한 모델을 쓸 수 있습니다. 보통 데이터 포인트가 많을수록 다양성도 높아지므로, 데이터셋이 클수록 더 복잡한 모델을 만들 수 있습니다. 단, 동일한 데이터를 단순히 복제하거나 매우 비슷한 데이터를 추가하는 것은 도움이 되지 않습니다.

배를 구매하고 싶어하는 고객을 예측하는 예시로 돌아가봅시다. 만약 "45세 이상이고 자녀가 3명 미만이거나 이혼하지 않은 고객은 배를 사고 싶어 한다"는 규칙을 12행짜리 데이터에서 도출했다면, 그 규칙을 얼마나 신뢰할 수 있을까요? 하지만 같은 규칙이 10,000명의 고객 데이터에서도 성립한다면 이야기가 달라집니다.

데이터가 많을 때

더 복잡한 모델도 일반화가 잘 됩니다. 데이터 포인트가 많다는 것은 모델이 학습할 패턴이 충분하다는 의미이고, 개별 포인트에 과도하게 맞춰질 위험이 줄어듭니다.

데이터가 적을 때

단순한 모델이 더 안전합니다. 데이터가 적을수록 복잡한 모델은 그 데이터를 단순히 외워버릴 가능성이 높고, 일반화 성능은 오히려 떨어집니다.

→ 모델 복잡도 제한 필요

더 많은 데이터를 확보하고 그에 맞게 더 복잡한 모델을 만드는 것은 지도 학습 성능을 높이는 데 종종 큰 효과를 발휘합니다. 현실에서는 수집할 데이터의 양을 결정할 수 있는 경우가 많은데, 이것이 모델을 이리저리 조정하는 것보다 더 유익할 수 있습니다. 데이터의 힘을 과소평가하지 마세요.
