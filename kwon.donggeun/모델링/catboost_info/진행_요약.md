# OTT 이탈 예측 모델링 진행 요약

## 프로젝트 개요

- **목표**: OTT 멤버십 데이터 기반 고객 이탈(churn) 예측
- **타겟**: `repurchase` — 재결제(1) / 이탈(0)
- **클래스 비율**: 재결제 65.6% / 이탈 34.4%
- **최종 데이터**: 17,774행 × 45 피처 (피처추가본 v2)

---

## 데이터 발전 흐름

| 단계 | 파일 | 행수 | 피처수 | 주요 내용 |
|------|------|------|--------|-----------|
| 원본 | `Membership.csv` | 18,183 | 10 | 결측치·이상값 미처리, 혼합 타입 |
| 전처리본 | `Membership_processing.csv` | 17,774 | 24 | 중복 제거, 인코딩, 결측치 처리, View_History 집계 피처 추가 |
| 피처추가본 | `Membership_v2.csv` | 17,774 | 45 | 파생 피처 추가 (야간가입, 요일, 프로모션 교호작용 등) |

### 전처리 주요 작업
- `user_no` 중복 338건 → `duration_days` **최댓값** 행 유지, 나머지 삭제
- `concurrent_streams=3` 7건 → 오입력으로 판단, 제거
- `repurchase / promotion_yn / is_churn_prevented`: O→1, NaN→0
- `gender=N` → F/M 비율 기반 랜덤 대체
- `age=40` 미인증 기본값 3,590건 → 인증유저 분포로 재배분
- `amount` 달러($) → 원화(₩) 통일
- View_History 집계 6개 피처 추가 (`total_watch_count`, `total_watch_duration`, `unique_movies`, `avg_duration`, `watch_days_count`, `has_watch_history`)

## 모델링 진행 흐름

### 1단계 — 기본 모델 비교 (섹션 3)

5개 모델 × 3개 데이터셋 비교. **Train / Test 동시 출력으로 과적합 여부 확인.**

| 모델 | 원본 Test AUC | 전처리본 Test AUC | 피처추가본 Test AUC | 과적합 |
|------|--------------|-----------------|-------------------|--------|
| Logistic Regression | 0.6225 | 0.6488 | 0.6487 | 없음 ✅ |
| **Gradient Boosting** | **0.6667** | **0.6693** | 0.6632 | 없음 ✅ |
| LightGBM | 0.6439 | 0.6481 | 0.6532 | 심함 ❌ |
| XGBoost | 0.6262 | 0.6136 | 0.6286 | 심함 ❌ |
| Random Forest | 0.6039 | 0.6420 | 0.6469 | 매우 심함 ❌ (Train≈1.0) |

> **핵심 발견**: LightGBM/XGBoost/RF는 기본 파라미터에서 과적합 심각 → Optuna 튜닝 필요

출력 형식:
```
모델                    구분    Accuracy   F1   Precision  Recall  ROC-AUC  PR-AUC
Logistic Regression     Train   0.6875  0.8039  0.6927  0.9547  0.6446  0.7626
                        Test    0.6948  0.8077  0.6963  0.9616  0.6487  0.7652  Gap:-0.0041
```

## 지표 설명

| 지표 | 설명 |
|------|------|
| **ROC-AUC** | 이탈/재결제 구분 능력. 0.5=랜덤, 1.0=완벽. 핵심 지표 |
| **PR-AUC** | 불균형 데이터에서 더 엄격한 AUC. 기준선=양성비율(0.34) |
| **Accuracy** | 전체 정확도. 불균형 데이터에선 단독으로 신뢰 어려움 |
| **F1** | Precision × Recall 조화평균. 불균형 데이터에서 Accuracy 보완 |
| **Precision** | 이탈 예측 시 실제 이탈 비율. 오경보 최소화 |
| **Recall** | 실제 이탈 중 잡아낸 비율. 이탈 고객 누락 최소화 |
| **Gap** | Train AUC − Test AUC. >0.05면 과적합 의심 |
