컬럼명타입설명처리 방법
product_cd str 구독 상품 코드 (pk_xxxx) 원본 09.모델링에서 LabelEncoder
billing_method int 결제 방식 코드 원본
amount_krw float 결제 금액 (원화 환산). USD 결제자는 × 1100 변환 후 19,349원 상한 처리 파생 amount가 소수면 USD → ×1100 / 정수면 KRW 그대로
is_usd int (0/1) USD 결제 여부 파생 amount % 1 ≠ 0 이면 1
concurrent_streams float 동시 접속 가능 스트림 수 (1/2/4) 원본 결측치 73건 → 가격대별 최빈값으로 대체
promotion_yn int (0/1) 프로모션 적용 여부 인코딩 O=1, N/NaN=0
is_churn_prevented int (0/1) 이탈 방지 조치 시행 여부 (조치를 취한 것이지 이탈 막힌 것 아님) 인코딩 O=1, N/NaN=0
payment_device str 결제 기기 (android / ios / pc / lgtv / sstv / ott / smarttv / mobile) 원본 09.모델링에서 LabelEncoder
is_user_verified int (0/1) 본인인증 여부 인코딩 Y=1, N/NaN=0
gender int (0/1) 성별. F=0, M=1 인코딩 미인증(N) → 인증 유저 F/M 비율로 랜덤 대체 후 0/1 인코딩
age float 나이 (세). 10세 미만·100세 이상 제거 원본 결측치 155건 → 인증 유저 중앙값(30세)으로 대체
reg_hour int 가입 시간 (0~23시) 원본
sub_days int 구독 기간 (일). end_date - reg_date 파생 ⚠️ 이탈자는 sub_days 짧음 → 준-리케이지 가능성

🗑️ 제거 컬럼: amount (amount_krw로 대체), age_group (age로 대체), reg_date / end_date (sub_days 계산 후 불필요), user_no / uid / USER_ID (식별자)

📺 시청 행동 피처
컬럼명타입설명생성 방법
watch_count float 총 시청 세션 수 (동일 영화 여러 번 포함) 파생 View_History 건수 집계
unique_movies float 시청한 고유 영화 수 파생 MOVIE_ID nunique
active_days float 시청한 날 수 (접속일 수) 파생 WATCH_DAY nunique
total_duration float 총 시청 시간 (분) 파생 DURATION 합계
avg_duration float 세션당 평균 시청 시간 (분) 파생 DURATION 평균
avg_completion float (0~1) 영화별 평균 완주율 (1.0 = 끝까지 시청) 파생 영화 길이 대비 시청 시간 비율, clip(1.0)
avg_rewatch_ratio float (≥0) 평균 시청 배율 (1.0 초과 = 재시청 있음) 파생 영화 길이 대비 시청 시간 비율, clip 없음
rewatch_count float 재시청한 영화 수 (watch_ratio > 1.0인 영화 수) 파생
watch_per_day float 구독 1일당 시청 세션 수 파생 watch_count / sub_days (sub_days=0이면 1로 처리)

시청 기록 없는 유저(미시청) → 모든 시청 피처 0으로 처리

🚀 가입 초기 행동 피처
컬럼명타입설명생성 방법
days_to_first_watch float 가입 후 첫 시청까지 걸린 일수. 미시청=-1 파생 첫 WATCH_DAY - reg_date
cold_start int (0/1) 가입 후 7일 이내 첫 시청 여부 (=초기 참여율) 파생 days_to_first_watch ≤ 7 이면 1
📊 RFM 피처
컬럼명타입설명생성 방법
recency_days float 마지막 시청 후 경과 일수 (R). 미시청 유저=최대값+1 파생 기준일(최대 WATCH_DAY) - 마지막 시청일

F(Frequency) = watch_count, M(Monetary) = amount_krw 로 대체 사용
